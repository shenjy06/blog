CSS 属性书写顺序最佳实践

一、推荐的属性书写顺序

大厂（Google、Airbnb、Bootstrap 等）普遍遵循的顺序：

1. 定位属性（Positioning）

影响元素在文档流中的位置

2. 盒模型属性（Box Model）

元素自身的尺寸和空间

3. 排版属性（Typography）

文本相关样式

4. 视觉属性（Visual）

背景、颜色等视觉效果

5. 其他属性（Misc）

动画、过渡等

五、记忆口诀

“定盒排视其”

  1. 定位（Positioning）- position, top, z-index
  2. 盒模型（Box Model）- display, width, padding, margin, border
  3. 排版（Typography）- font, color, text-align
  4. 视觉（Visual）- background, shadow, opacity
  5. 其他（Others）- transform, transition, animation

六、实际工作建议

✅ 推荐做法

  • 使用 Stylelint + 自动格式化工具
  • 团队统一配置文件
  • 代码审查时检查顺序
  • 使用 CSS-in-JS 时也遵循顺序

⚠️ 注意事项

  • 不必过度纠结，保持一致性最重要
  • 简写属性（如 background）放在对应分类中
  • 浏览器前缀属性紧跟标准属性后面
  • 同类属性按字母排序（可选）

📋 团队协作

总结：大厂通用顺序是 定位 → 盒模型 → 排版 → 视觉 → 其他，最重要的是团队统一并使用自动化工具强制执行。
